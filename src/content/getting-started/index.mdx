# Getting Started with Raiken

Welcome to Raiken, a modern, AI-powered web application testing tool that revolutionizes how developers create, manage, and execute automated tests. By combining the power of Playwright browser automation with advanced AI capabilities, Raiken provides an intuitive visual interface for generating comprehensive test suites from natural language descriptions.

## What is Raiken?

Raiken is a comprehensive testing platform that bridges the gap between manual testing and automated test creation. It consists of two main components:

1. **Web Application**: A Next.js-based interface for visual test creation and management
2. **CLI Tool**: A command-line interface that integrates directly with your local development environment

## Prerequisites

Before getting started, ensure you have:

- **Node.js** 16.x or later
- **npm** or **yarn** package manager
- A modern web browser
- Basic knowledge of JavaScript/TypeScript

## Quick Installation

Choose your preferred installation method:

### Option 1: CLI Tool (Recommended)

```bash
# Install globally
npm install -g @raiken/cli

# Navigate to your project
cd my-nextjs-app

# Start Raiken (auto-detects project type)
raiken start
```

### Option 2: Standalone Application

```bash
git clone https://github.com/your-username/raiken.git
cd raiken
npm install
echo "OPENROUTER_API_KEY=your_api_key_here" > .env.local
npm run dev
```

## Your First Test

Once Raiken is running, you can create your first test:

1. **Open the web interface** at `http://localhost:3460`
2. **Describe your test** in natural language:
   ```
   Test the shopping cart checkout process
   ```
3. **Generate the test** using AI
4. **Review and customize** the generated test
5. **Run the test** across multiple browsers

### Key Components

#### 1. **Web Application Layer**
- **Next.js Framework**: Modern React-based web application with App Router
- **Component Architecture**: Modular UI components using Radix UI and Tailwind CSS
- **State Management**: Zustand for efficient, type-safe state management
- **Real-time Updates**: Live DOM extraction and browser status monitoring

#### 2. **Service Layer**
- **AI Service**: Abstraction layer for AI-powered test generation using OpenRouter
- **Browser Service**: High-level browser automation operations
- **Test Service**: Test lifecycle management (generation, execution, storage)
- **API Service**: Centralized HTTP client with error handling and retry logic

#### 3. **Core Engine**
- **Playwright Service**: Browser automation engine supporting multiple browsers
- **DOM Analyzer**: Intelligent DOM tree extraction and element selection
- **Test Script Utils**: Utilities for test generation and execution

#### 4. **CLI Integration**
- **Project Detection**: Automatic framework detection (Next.js, React, Vue, etc.)
- **File System Bridge**: Direct integration with local project files
- **Configuration Management**: Project-specific settings and preferences
